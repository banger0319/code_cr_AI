# Web Blocking Conditions

以下情况必须标记为 `Blocking: true`：

- 新增或暴露密钥、token、密码、私钥、内部服务地址等敏感信息。
- 删除、绕过或弱化登录、鉴权、权限校验、租户隔离。
- 引入明确的 XSS、SQL 注入、命令注入、路径穿越、SSRF 等高危漏洞。
- 新增 `eval`、`new Function`、不可信 HTML 注入，且输入来源不可控或未净化。
- 生产代码使用 mock、debug bypass、硬编码管理员身份或绕过真实接口。
- 删除关键错误处理，可能导致支付、订单、权限、数据一致性等核心链路异常。

以下情况通常标记为 `Blocking: false`：

- 仅代码风格、命名、轻微可读性问题。
- 可优化但不影响正确性和安全性的性能建议。
- 无法仅基于 diff 和仓库上下文确认的推测性问题。
